API Test Server

    Server Specification

  • https://github.com/taejukim/django_server
    • Specification
      • Docker
      • Python 3.9(Slimbuster)
      • Django 3.1
      • Nginx

    • HTTP Method Test

  • /api/method/
    • 해당 URL로 각 Method 요청
    • GET, POST, PUT, PATCH, DELETE, COPY, HEAD, OPTIONS, LINK, UNLINK, PURGE, LOCK, UNLOCK, PROFIND, VIEW 지원 가능
    • 필수 : 해당 URL은 "/" 로 끝나야함

    • HTTP Path Test

  • /api/path/
    • 해당 URL에 임의 Path를 요청
    • 경로 깊이 및 Type은 제한이 없으며 요청한 URL Path 를 리턴함
    • e.g.)http://133.186.210.236/api/path/1/abc/def/3/4
    • GET, POST, PUT, PATCH, DELETE, COPY, HEAD, OPTIONS, LINK, UNLINK, PURGE, LOCK, UNLOCK, PROFIND, VIEW 지원 가능

    • HTTP Status code Test

  • /api/status
    • code 파라미터로 Status Code 전달
    • e.g.)http://133.186.210.236/api/status?code=404
    • https://developer.mozilla.org/ko/docs/Web/HTTP/Status에 표시된 모든 Http Status Code Test 가능

    • HTTP 응답 시간 Test

  • /api/delay
    • second 파라미터로 Delay 시간 전달(초)
    • http://133.186.210.236/api/delay?second=10
    • 소수로도 전달 가능 (e.g. http://133.186.210.236/api/delay?second=30.5

    • HTTP Content Type Test

  • /api/contents
    • type 파라미터로 Contents Type 전달 (html, xml, json)
    • e.g.)http://133.186.210.236/api/contents?type=xml

    • 서버 장애 Test

  • /api/server_failure/
    • 서버 장애 발생 Test
    • 서버의 HTTP 응답 코드를 조정하여 장애 발생
    • 서버의 응답 시간을 조절하여 Time out 장애 발생
    • 장애 설정 Page /api/server_failure/adjust

    • Big size response body Test

  • /api/bigbody/
    • 큰 크기의 Response body를 요청
    • GET /api/bigbody/?bytes=[size] 와 같이 응답 받고자 하는 body 에 size를 byte 단위로 입력

    • File upload Test

  • /api/file-upload/
    • Post 요청 Multipart/data 로 file 업로드 시 업로드 된 파일의 이름과 Size를 반환
